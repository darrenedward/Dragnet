## Risks / watch-outs

- **Fallback semantics:** do not reintroduce fallback-on-quality-failure while adding telemetry, breaker, or resume.
- **Failed-run persistence:** failed scans still need telemetry when usage was captured.
- **Custom endpoints:** some OpenAI-compatible endpoints omit `response.usage`; handle null usage honestly.
- **SDK signal shape:** verify cancellation with TypeScript and a manual abort test.
- **Message serialization size:** checkpoint files can grow; if they exceed roughly 1MB in real scans, truncate older tool outputs while preserving recent context.
- **Stale-run reaping:** current auto-reap behavior must be changed carefully so stale runs with no checkpoint still recover cleanly.
- **Per-chunk breaker counting:** count per chunk unless product direction changes; a provider failing every chunk is the exact behavior the breaker should catch quickly.
- **Dirty worktree:** preserve unrelated local edits while implementing.
